线程(Thread)的五个状态：
新建，就绪，阻塞，运行，死亡
New，Runnable，Running，Blocked，Dead

  • 新建：new Thread()之后，尚未执行start()方法时，线程处于此状态。
  • 就绪：执行start()之后，还未竞争到CPU；

  • 一个时刻只有一个线程处在运行状态，其余线程均处于就绪状态；这些就绪状态的线程是由Java运行时系统的线程调度程序(thread scheduler)来调度的。 linkto: thread scheduler

  • 阻塞：有多种可能原因使线程进入阻塞：

    1. sleep()
    2. I/O request
    3. 等待同步锁
    4. 等待触发条件 etc.
  • 运行：获得CPU时间片，真正开始执行run()
  • 死亡：run()正常退出或是被异常中断，判断方法为isAlive()

线程的同步方法：

  1. 同步方法

    • synchronized修饰普通方法 -> java每个对象都有一个内置锁，当用此关键字修饰方法时，内置锁会保护整个方法；<内置锁：英语？内存模型？>
    • 经测试，synchronized修饰普通方法时，所有方法会使用同一个内置锁，因此会互斥。
      即synchronized method1()和
      synchronized method2() 不能同时运行；
    • synchronized修饰静态方法 -> 会锁住整个类。
    • synchronized原理：
      • 反编译：javap -c
      • monitorenter - monitorexit

  2. 同步代码块

    • 如果锁是对象，则只对当前对象加锁；
    • 如果锁是类，则对当前类及所有对象加锁；

  3. wait-notify
    几个要点：

    • wait和notify调用者都是锁，并且调用时必须持锁，否则会抛出 java.lang.IllegalMonitorStateException异常;
    • join和yield调用对象都是Thread;

  4. volatile

    • About Java memory:
      • 主存 - 内存
      • CPU高速缓存
    • 为了CPU运行速度考虑，不会每次CPU运算都去主存（内存）读写。
    • 对单核CPU也会有类似的问题，只是以线程调度的形式分别执行的<何种线程调度形式？>
    • 并发编程中的三个概念：
      • 原子性：一个操作或多个操作，要么全部执行，要么全不执行；
      • 可见性：一个线程修改了变量值，其他线程立即能看到；
      • 有序性：按照代码的先后顺序执行。实际不一定，可能会发生指令重排序（instruction reorder）——目的：为了提高运行效率。linkto：指令重排序
      • 扩展：linkto：JSR133、双重检查模式（double-check idiom）

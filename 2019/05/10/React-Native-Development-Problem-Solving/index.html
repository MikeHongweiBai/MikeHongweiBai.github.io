#####

Npm Install Failure

While run: npm install in project root directory

OS: Windows 10

Error:

Conclusion:

modify package.json file, change the version of conventional-changelog to a valid one. (@1.2.1 in this case)

Then run npm install again.

Also, I added Windows Environment Variables, but not sure it helped:

Analysis:

Literally it said that npm could not find this dependency: *conventional-changelog@1.2.0*

So Google it, I found its github page,

and dependency import document, which indicating that 1.2.1 is the latest version.

So change it and try again.

SDK location not found while run: react-native run-android

While run: react-native run-android in project root.

OS: Windows 10

Error:

SDK location not found. Define location with sdk.dir in the local.properties file or with an ANDROID_HOME environment variable.

Conclusion:

added Windows Environment Variables:

Replace your \<USERNAME> with your own in Windows OS.

And If you run application in Android Studio, it may not raise this error.

Invariant Violation: View config not found for name div. Make sure to start component names with a capital letter.

While compile React-Native project.

OS: Windows 10

Error:

Conclusion:

HTML tags such as: div, ol etc. could not be used in React-Native while it runs on a mobile device. (It could run on a web)

  • use View instead of div;

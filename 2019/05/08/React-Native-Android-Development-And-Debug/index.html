React Native (Android) Development And Debug

2019-05-08
#Android #React Native #Guide
React Native/ Note

First thing is to find a comfortable way of coding. :D

We have more than one choices:

  • Android phone physically.
  • Android Emulator by Android Studio (AVD)
  • Other 3rd party Android Emulator

I never try the 3rd option, and it is not necessary. So ignore it.

Let’s try option 1 and 2.

As an Android developer, of course I have more than one Android device. :D

I am so exciting because it finally enters the world I could dominate - coding. I do not like environment establishment stuff, it is necessary though.

I assume that you have followed my previous posts:

  • Start A React Project with npm Way

  • Start A React Hello World

If not, and you could not follow, read them first.

Physical Android Device.

Steps:

  1. connect your Android device (normally a phone) with USB cable. (Type-C or others)

  2. Open PowerShell (Windows) or terminal (MacOS or Unix), run

    ensure that your device are available on your PC.

    If not, please Google to search a solution.

  3. Go to your project root, and execute:

    It will automatically install and launch react native application on your phone.

  4. The next step is to modify some code on your project, and see the changes on your phone.

    Open VSCode or any IDE you preferred to manage your react code. (Not Android Studio)

    Change something easy to tell, for instance, change some text.

  5. Then press menu button on your Android phone (my Huawei phone does not have this button) or shake your phone 3-4 times.

    A menu will pop up with the first item: Reload.

    Press it and wait…

    Normally you may got error and not what you expected. No worry.

    I have the same problem, as shown below:

  1. Just follow its instruction to connect to the react native server on PC.

    There are two ways, a) Wifi b) adb reverse. Note that adb reverse is introduced from Android 5.0. If your Android device is lower than that, it raises error.

  2. Now change your code, then reload. It works.

Note:

You could send ADB command to simulate a menu click (or shake it :/)

  2. It connects to your react native server automatically.

How to see console.log()

Pre-requirements recap:

Make sure:

  1. Open Android Studio or run adb devices.
  2. run react-native run-android first.

Steps:

  1. Send adb shell input keyevent 82 to show debug menu and select: Debug JS Remotely:

    !rn_debug_menu1

  2. Send adb shell input keyevent 82 to show debug menu again and select: Reload.

  3. Open url: http://localhost:8081/debugger-ui in your browser. I use Chrome for example.

  4. Actually you could not see log here. You should open Chrome developer tools use shortcut: Ctrl+Shift+I or GUI as below:

  5. Then you get log output on the right side:

  6. Here you could see all logs, by console.log(), console.warn() etc.
